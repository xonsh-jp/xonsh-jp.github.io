静的設定ファイル¶

実行制御ファイルに加えて、xonshには静的な設定ファイルがあります。このJSON形式のファイル$XONSH_CONFIG_DIR/config.jsonは、通常は存在します~/.config/xonsh/config.json。このファイルの目的は、何か他のことが起こる前にランタイムパラメータを設定できるようにすることです。これには、さまざまな外部シェルからのデータのロードや重要な環境変数の設定が含まれます。

これはトップレベルの辞書またはJSONオブジェクトです。これには、次の最上位キーがあります。すべてのトップレベルキーはオプションです。

env¶

これは環境変数を設定するための簡単な文字列キー付き辞書です。例えば、

{ "env" ： { 
 "EDITOR" ： "xo" 、
 "PAGER" ： "more" 
 } 
}

xontribs¶

これは、実行制御ファイルをロードする前にロードするxontrib名（文字列）のリスト（JSON配列）です。例えば、

{ "xontribs" ： [ "mpl" 、 "example" ]}

foreign_shells¶

これは、環境変数、エイリアス、および外部シェル関数などの追加の起動情報を調べるために、外部シェルを表すdicts（JSONオブジェクト）のリスト（JSON配列）です。収集された一連のデータは、今後拡張される可能性があります。それぞれのシェル辞書は解凍され、xonsh.foreign_shells.foreign_shell_data()関数に渡されます。したがって、これらの辞書は次の構造を持ちます。

シェル：str、required - シェルの名前またはパス（ "bash"や "/ bin / sh"など）。
インタラクティブ：bool、optional - シェルを対話モードで実行するかどうか。 default=true
ログイン：bool、optional - シェルをログインシェルにするかどうか。 default=false
envcmd：str、optional - で環境出力を生成するコマンドです。 default="env"
aliascmd：str、optional - エイリアス出力を生成するコマンドです。 default="alias"
extra_args：strのリスト、オプション - シェルに渡す追加のコマンドラインオプション。default=[]
currenv：dictまたはnull、オプション - 現在の環境の手動オーバーライド。 default=null
安全：bool、optional - 例外やその他のエラーを安全に処理するかどうかを示すフラグ。default=true
prevcmd：str、optional - 他のコマンドより前に実行する追加のコマンドまたはスクリプト.Sourcingや環境リカバリが必要なその他のコマンドに便利です。default=''
postcmd：str、optional - 他のすべてのコマンドの後に実行するコマンドprevcmd。これが原因で発生した可能性のあるダメージをクリーンアップするのに便利です。default=''
funcscmd：strまたはNone、optional - これは、外部シェルにネイティブな関数の名前と場所を判別するために使用できるコマンドまたはスクリプトです。このコマンドは、関数が定義されている関数名とファイル名の組の空白で区切られたシーケンスのみを出力します。これがNone（null）の場合、デフォルトのスクリプトはシェル名に基づいて検索されようとします。これらの関数の呼び出し可能なラッパーはエイリアス辞書に返されます。default=null
ソース：strまたはNone、optional - そのシェルで関数を呼び出す目的で外部シェルファイルをソースする方法。これがNoneの場合、デフォルト値はシェル名に基づいて検索されます。default=null
runcmd：strまたはNone、optional - -cBashや/Ccmd.exe など、スクリプトの実行時に使用するコマンドラインスイッチ。 default=null
seterrprevcmd：strまたはNone、optional - 他のすべてのコマンドの前にシェルのexit-on-errorを有効にするコマンド。たとえば、これはBashの "set -e"です。このエラー終了時の動作を無効にするには、単に空の文字列を渡します。 default=null
seterrpostcmd：strまたはNone、optional - 他のすべてのコマンドの後にシェルのexit-on-errorを有効にするコマンド。たとえば、これは "errorlevel 1 exit 1" cmd.exeの場合です。このエラー終了時の動作を無効にするには、単に空の文字列を渡します。default=null

以下にいくつかの例を示します。

＃負荷のbashその後のzsh 
{ "foreign_shells" ： [ 
    { "シェル" ： "/ binに/ bashの" }、
    { "シェル" ： "zshの" } 
 ] 
}

＃ロードバッシュをカスタムrcfile 
{ "foreign_shells" ： [ 
    { "shell" ： "bash" 、
     "login" ： true 、
     "extra_args" ： [ "--rcfile" 、 "/ path / to / rcfile" ] 
     } 
 ] 
}

＃空のリスト
{ "foreign_shells" ： []}を介してすべての外部シェルの読み込みを無効にする

それをまとめて¶

次の例は、完全に構成された設定ファイルを示しています。

Download config.json

{ "env" ： { 
  "EDITOR" ： "xo" 、
  "PAGER" ： "more" 
  }、
 "xontribs" ： [ "mpl" ]、
 "foreign_shells" ： [ 
  { "シェル" ： "bash" 、
   "login" ： 真、
   "extra_args" ： [ "--rcfile" 、 "/パス/を/のrcfile" ] 
   }、
  { "シェル" ： "zshの" } 
  ] 
}
